Disaster Response and Recovery System

A collaborative web-based system designed to enhance disaster management through various modules tailored to specific user roles. The system includes incident reporting, donation management, assistance requests, and interactive hazard maps, all while enabling automated communication through SMS and email.

Tech Stack: PHP Laravel, MySQL, Leaflet.js, Twilio API, PHPMailer, HTML, CSS, JavaScript, Bootstrap

View on GitHub

DRRS Modules:

Admin:
- Manage patient care reports
- Manage incident reports
- Manage users
- Manage hazard map (pin the locations of hazards and shelters)

Responder:
- Create patient care reports
- Manage incident reports

Secretary:
- Manage donations
- Manage assistance requests

Resident:
- Request assistance/relief goods
- View hazard map
- Donate
- Report incidents and pin accident locations

Key Contributions:
- Integrated Twilio API for SMS notifications.
- Configured PHPMailer for password reset emails.
- Built interactive map with Leaflet.js to visualize accident prone area.
- Developed core modules for donations, incident reporting, and assistance requests.
- Implemented PDF report generation using Barryvdh DomPDF.
